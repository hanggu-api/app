// turbo-all
Este workflow garante que nenhuma funcionalidade seja desfeita por comandos futuros.

1. **ANTES de qualquer mudança**: Leia `requirements_checklist.md` na raiz do projeto para entender as regras de UI e funcionalidades críticas.
2. **DURANTE a implementação**: Certifique-se de que o novo código não quebra nenhum item do checklist.
3. **APÓS a conclusão**: Atualize o `requirements_checklist.md` se novas regras forem criadas ou se itens pendentes forem resolvidos.
4. **Persistência**: Nunca remova itens do checklist sem aprovação explícita do usuário.
